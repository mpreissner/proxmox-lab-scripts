# Code of Conduct

## Our Pledge

This project exists to help security engineers and homelab enthusiasts build realistic testing environments. We're committed to keeping it a welcoming, collaborative space for everyone — regardless of experience level, background, or the specific security products you work with.

## Our Standards

**Encouraged behaviors:**

- Sharing lab configurations, traffic patterns, or integration ideas that others might find useful
- Asking questions, even basic ones — everyone starts somewhere
- Giving constructive feedback on pull requests and issues
- Acknowledging when you're not sure about something

**Not acceptable:**

- Dismissive or condescending responses to questions
- Personal attacks or harassment of any kind
- Using this project's issues or discussions to promote commercial products
- Sharing configurations or modifications intended to harm real systems or users

## Scope

This code of conduct applies to all project spaces: issues, pull requests, discussions, and any other communication directly related to this repository.

## Enforcement

If you experience or witness unacceptable behavior, please reach out via the repository's [Security Advisory](../../security/advisories/new) feature (for privacy) or flag the content using GitHub's built-in reporting tools.

Maintainers reserve the right to remove comments, close issues, or restrict participation from anyone who violates these standards.
